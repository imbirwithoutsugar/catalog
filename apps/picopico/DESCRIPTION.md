# PicoPico для Лілки

Це - порт проєкту PicoPico для Лілки на ESP32-S3. PicoPico - це плеєр для ігор у форматі Pico-8.

> **Pico-8** є власністю **Lexaloffle Games**. Цей проєкт є неофіційним плеєром і не пов'язаний з Lexaloffle.

## Особливості

- Відтворення ігор у форматі Pico-8 (`.p8`, текстовий формат)
- Завантаження картриджів зі SD-картки з теки `/Pico8/`
- Вбудовані ігри прошиті в бінарник (якщо SD-картки немає або тека порожня)
- Зображення масштабується до 240×240 на дисплеї Лілки (нативно Pico-8 — 128×128)
- Підтримка графіки, звуку та керування
- Оптимізовано для ESP32-S3 з PSRAM (буфери картриджів і кадру виділяються в PSRAM)
- Звук через I2S

## Завантаження ігор зі SD-картки

1. Створіть на SD-картці теку з назвою `Pico8` у корені (саме з великої `P`):
   ```
   /Pico8/
   ```
2. Покладіть у неї картриджі у **текстовому** форматі `.p8` (не `.p8.png`).
   Завантажити їх можна, наприклад, з [BBS Lexaloffle](https://www.lexaloffle.com/bbs/?cat=7) — кнопка «Cart» → файл `.p8`.
3. Вставте картку в Лілку та запустіть PicoPico. Перші **32** знайдені картриджі з'являться у меню вибору.

Якщо тека `/Pico8` відсутня або в ній немає `.p8` файлів — PicoPico завантажить вбудовані ігри.

> Розмір одного картриджа обмежено **256 КБ** (більш ніж достатньо для будь-якого офіційного `.p8`).

## Керування

В **меню вибору картриджа**:

| Кнопка | Дія |
|---|---|
| ↑ / ↓ | Вибір картриджа |
| A або B (або C / D) | Запустити вибраний картридж |

У **грі**:

| Лілка | Pico-8 |
|---|---|
| D-Pad ←→↑↓ | D-Pad (кнопки 0-3) |
| A | 🅾️ (кнопка 4, `btn(4)` / Z на PC) |
| C | 🅾️ (дублює A) |
| B | ❎ (кнопка 5, `btn(5)` / X на PC) |
| D | ❎ (дублює B) |
| **Start** або **Select** | Вийти з гри до меню вибору картриджа (виконується перезавантаження для чистого старту) |

> Pico-8 підтримує лише 6 кнопок на гравця — кнопки A/C і B/D навмисно продубльовані, щоб гра була зручнішою незалежно від розташування пальців.

## Збірка з вихідного коду

Для збірки потрібен PlatformIO CLI або VS Code з розширенням PlatformIO.

```bash
cd lilka
pio run
pio run --target upload
```

## Корисна інформація

- Якщо гра «зависла» або поводиться некоректно — натисніть **Start** або **Select**, щоб повернутися до меню.
- Деякі картриджі використовують API, які ще не повністю реалізовані: `fillp()`, `music()` (заглушка), частина ефектів палітри. Такі ігри можуть виглядати неправильно, але зазвичай запускаються.
- Картриджі у форматі `.p8.png` **не** підтримуються — конвертуйте їх у `.p8` через офіційний Pico-8 (`SAVE filename.p8`) або онлайн-конвертери.
- Файли з не-латинськими символами в назві можуть відображатись некоректно у меню — рекомендовано називати картриджі латиницею.

## Відомі обмеження

- `fillp()` (візерункова заливка) — не реалізовано.
- `music()` — заглушка, фонова музика не грає (SFX працюють).
- Деякі ігри з нестандартним використанням палітри можуть мати глюки кольорів.
- Тільки гравець 1; `btn(b, 1)` та далі завжди повертають `false`.
