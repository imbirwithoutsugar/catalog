# PicoPico для Лілки

Це — порт проєкту PicoPico для Лілки на ESP32-S3. PicoPico — це плеєр для ігор у форматі Pico-8.

> **Pico-8** є власністю **Lexaloffle Games**. Цей проєкт є неофіційним плеєром і не пов'язаний з Lexaloffle.

## Особливості

- Відтворення ігор у форматі Pico-8 (`.p8`, текстовий формат)
- Завантаження картриджів зі SD-картки з теки `/Pico8/`
- Вбудовані ігри прошиті в бінарник (якщо SD-картки немає або тека порожня)
- Зображення масштабується до 240×240 на дисплеї Лілки
- Підтримка графіки, звуку та керування
- Оптимізовано для ESP32-S3 з PSRAM
- Звук через I2S

## Завантаження ігор зі SD-картки

1. Створіть теку `Pico8` в корені SD-картки (з великої `P`)
2. Покладіть у неї картриджі у **текстовому** форматі `.p8` (не `.p8.png`). Завантажити можна з [BBS Lexaloffle](https://www.lexaloffle.com/bbs/?cat=7) — кнопка «Cart» → файл `.p8`
3. Вставте картку в Лілку та запустіть PicoPico. Перші **32** знайдені картриджі з'являться у меню

## Керування

В **меню вибору картриджа**:

| Кнопка | Дія |
|---|---|
| ↑ / ↓ | Вибір картриджа |
| A або B | Запустити вибраний картридж |

У **грі**:

| Лілка | Pico-8 |
|---|---|
| D-Pad ←→↑↓ | D-Pad |
| A / C | 🅾️ |
| B / D | ❎ |
| Start або Select | Вийти до меню |

## Збірка

```bash
cd lilka
pio run
pio run --target upload
```

## Відомі обмеження

- `fillp()` — не реалізовано
- `music()` — заглушка, фонова музика не грає (SFX працюють)
- Картриджі `.p8.png` не підтримуються
